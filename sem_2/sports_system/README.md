# FC Bayern Munich Sports System  
### Лабораторная работа №2

## О проекте

**FC Bayern Munich Sports System** — это настольное приложение на Python с графическим интерфейсом, разработанное в рамках лабораторной работы.  
Программа предназначена для **учёта спортсменов**, их просмотра, поиска, удаления, хранения в базе данных и обмена данными через XML.

В качестве предметной области выбрана тематика **FC Bayern Munich**, поэтому все тестовые данные, интерфейс и демонстрационные XML-файлы оформлены в стилистике спортивной структуры клуба.

Проект реализован как **оконное приложение** с использованием **PySide6** и построен по архитектурному принципу **MVC (Model–View–Controller)**.

---

## Цель лабораторной работы

Цель работы — разработать оконное приложение, которое:

- работает с набором записей предметной области;
- использует **главное окно** и **дочерние диалоги**;
- поддерживает **поиск**, **удаление**, **добавление** и **просмотр** записей;
- хранит данные в **базе данных**;
- поддерживает **экспорт и импорт XML**;
- использует **DOM** для записи XML и **SAX** для чтения XML;
- реализует **табличное** и **древовидное** представление данных;
- использует **постраничный вывод**;
- содержит **menu bar** и **tool bar** с дублированием основных команд.

## Основной функционал

### 1. Просмотр записей
Программа позволяет просматривать спортсменов:
- в виде **таблицы**
- в виде **дерева**

### 2. Добавление записей
Через отдельный диалог можно добавить нового спортсмена в систему.

### 3. Поиск
Поиск реализован в отдельном диалоге с поддержкой:
- поиска по ФИО или виду спорта;
- поиска по диапазону титулов;
- поиска по ФИО или разряду.

Результаты поиска отображаются в самом диалоговом окне.

### 4. Удаление
Удаление реализовано через отдельный диалог по тем же условиям, что и поиск.

### 5. Постраничный вывод
В приложении реализована пагинация:
- первая страница;
- предыдущая страница;
- следующая страница;
- последняя страница;
- выбор количества записей на странице;
- отображение текущей страницы, общего количества страниц и общего числа записей.

### 6. Работа с базой данных
Все записи хранятся в **SQLite**.

### 7. Работа с XML
Поддерживаются:
- **сохранение XML через DOM**
- **загрузка XML через SAX**

### 8. Импорт и экспорт данных
Пользователь может:
- сохранить текущие записи в XML;
- загрузить записи из подготовленного XML-файла;
- полностью обновить содержимое базы данных данными из XML.

## Описание ключевых модулей

### `main.py`
Точка входа в приложение.  
Создаёт `QApplication`, подключает базу данных, создаёт репозиторий и запускает главный контроллер.

### `database/db_manager.py`
Отвечает за:
- подключение к SQLite;
- создание таблицы `sportsmen`;
- закрытие соединения.

### `models/sportsmen.py`
Содержит модель одной записи о спортсмене в виде `dataclass`.

### `models/enums.py`
Содержит перечисления:
- `SquadStatus`
- `SportRank`

### `models/repository.py`
Один из ключевых файлов проекта.  
Реализует:
- добавление записей;
- чтение всех записей;
- постраничное чтение;
- поиск;
- удаление;
- получение уникальных видов спорта и разрядов;
- полную замену содержимого базы.

### `controllers/main_controller.py`
Главный координатор логики приложения.  
Отвечает за:
- загрузку данных в интерфейс;
- пагинацию;
- открытие диалогов;
- переключение таблицы и дерева;
- импорт/экспорт XML;
- реакцию на действия пользователя.

### `views/main_window.py`
Главное окно приложения.  
Содержит:
- menu bar;
- tool bar;
- таблицу;
- дерево;
- кнопки команд;
- виджет пагинации;
- строку состояния.

### `views/add_dialog.py`
Диалог добавления записи.

### `views/search_dialog.py`
Диалог поиска с собственной таблицей результатов и отдельной пагинацией.

### `views/delete_dialog.py`
Диалог удаления записей.

### `services/tree_builder.py`
Преобразует список спортсменов в древовидную модель для отображения в `QTreeView`.

### `services/xml_dom_writer.py`
Сохраняет список спортсменов в XML через **DOM**.

### `services/xml_sax_reader.py`
Читает список спортсменов из XML через **SAX**.

### `views/widgets/pagination_widget.py`
Переиспользуемый виджет пагинации для главного окна и диалога поиска.

### `views/theme.py`
Содержит общую тёмную тему интерфейса с красными акцентами в стилистике FC Bayern Munich.

## Работа с XML

### Сохранение в XML
При сохранении используется **DOM**:
- создаётся XML-документ;
- для каждого спортсмена создаётся узел `<sportsman>`;
- вложенные поля сохраняются как дочерние элементы.

### Загрузка из XML
При загрузке используется **SAX**:
- XML читается потоково;
- данные собираются по мере чтения тегов;
- на основе прочитанных значений создаются объекты `Sportsman`.

### Почему это важно
Это не просто техническая деталь, а один из ключевых пунктов лабораторной работы:
- **DOM** демонстрирует построение XML-документа;
- **SAX** демонстрирует событийный разбор XML.

- открыть диалог **Добавить**;
- заполнить поля;
- сохранить запись;
- запись появится в таблице и в базе данных.

- выбрать команду **Загрузить XML**;

## Что особенно важно в этой работе

Ниже перечислены сильные стороны проекта, которые стоит выделять при защите:

### 1. Архитектура MVC
Приложение не написано “в одном файле”, а разбито на слои:
- данные;
- интерфейс;
- управляющая логика.

### 2. Отдельные диалоги
Поиск, удаление и добавление реализованы через отдельные окна, как и требуется в задании.

### 3. XML через разные подходы
- запись: **DOM**
- чтение: **SAX**

Это один из самых важных технических пунктов лабораторной работы.

### 4. База данных + XML
Проект сочетает:
- постоянное хранение в БД;
- импорт и экспорт внешних файлов XML.

### 5. Два представления данных
- таблица
- дерево

### 6. Пагинация
Реализована не формально, а в полноценном виде.

### 7. Тематическое оформление
Интерфейс и данные выдержаны в тематике **FC Bayern Munich**, что делает проект более цельным и аккуратным.
