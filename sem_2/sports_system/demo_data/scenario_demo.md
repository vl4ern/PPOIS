# Сценарий демонстрации лабораторной работы

## 1. Запуск программы
- Запустить `main.py`.
- Показать главное окно приложения.
- Кратко объяснить, что проект выполнен в архитектуре MVC:
  - `models` — данные и репозиторий,
  - `views` — интерфейс,
  - `controllers` — связывающая логика.

## 2. Демонстрация главного окна
- Показать menu bar.
- Показать tool bar с дублированием основных команд.
- Показать таблицу записей.
- Переключить отображение на дерево и обратно на таблицу.
- Показать статусную строку.

## 3. Демонстрация пагинации в главном окне
- Переключить размер страницы: 5, 10, 20.
- Перейти на первую страницу.
- Перейти на следующую страницу.
- Перейти на последнюю страницу.
- Вернуться назад.
- Показать, что отображаются:
  - текущая страница,
  - общее число страниц,
  - число записей.

## 4. Добавление записи
- Открыть диалог добавления.
- Ввести новую запись о спортсмене.
- Сохранить запись.
- Показать, что запись появилась в БД и на последней странице таблицы.

## 5. Поиск
- Открыть отдельный диалог поиска.
- Показать 3 режима:
  1. По ФИО или виду спорта.
  2. По диапазону титулов.
  3. По ФИО или разряду.
- Выполнить поиск в каждом режиме.
- Показать, что результаты выводятся в самом диалоге поиска.
- Показать пагинацию внутри диалога поиска:
  - первая,
  - предыдущая,
  - следующая,
  - последняя,
  - изменение числа записей на странице.

## 6. Удаление
- Открыть отдельный диалог удаления.
- Показать 3 режима:
  1. По ФИО или виду спорта.
  2. По диапазону титулов.
  3. По ФИО или разряду.
- Выполнить удаление.
- Показать сообщение о количестве удалённых записей.
- Показать, что главное окно обновилось.

## 7. Работа с XML
- Через пункт меню или toolbar выбрать «Сохранить XML».
- Сохранить текущие данные в XML-файл.
- Объяснить, что запись XML реализована через DOM.
- Через пункт меню или toolbar выбрать «Загрузить XML».
- Загрузить один из подготовленных файлов:
  - `bayern_roster_football_60.xml`
  - `bayern_roster_multisport_60.xml`
  - `bayern_roster_extended_60.xml`
- Объяснить, что чтение XML реализовано через SAX.
- Показать, что БД полностью обновилась новыми данными после загрузки файла.

## 8. Демонстрация тестовых файлов
- Показать, что подготовлено несколько XML-файлов.
- В каждом файле не менее 50 осмысленных уникальных записей.
- Показать загрузку разных файлов и изменение содержимого таблицы.

## 9. Короткий итог для преподавателя
Сказать, что в работе реализованы:
- оконное приложение с главным окном и дочерними диалогами,
- архитектура MVC,
- хранение данных в БД,
- импорт/экспорт XML,
- DOM для записи XML,
- SAX для чтения XML,
- поиск и удаление по условиям варианта,
- таблица и дерево,
- постраничный вывод,
- menu bar и toolbar.
