# Документация по реализации ЛР №4

## Что было изменено

Исходная лабораторная работа №1 представляла собой настольное приложение с графическим интерфейсом на `customtkinter`.
Для выполнения требований ЛР №4 система была переработана в клиент-серверное веб-приложение на базе `Flask`.

## Общий код для ЛР №1 и ЛР №4

Общая бизнес-логика вынесена в отдельные модули:

- `app/models/` — модели предметной области.
- `app/services/parking_service.py` — вся логика работы парковки.
- `app/facade.py` — единая точка доступа для CLI и веб-интерфейса.

И CLI, и веб-интерфейс используют один и тот же код работы с моделью.
Это и демонстрирует требование о сохранении общего кода между лабораторными.

## Структура приложения

- `run_cli.py` — запуск консольного интерфейса.
- `run_web.py` — запуск веб-приложения.
- `app/web.py` — Flask-маршруты.
- `app/templates/` — HTML-шаблоны.
- `app/static/` — CSS-оформление.
- `data/parking_data.json` — сериализованное состояние модели.
- `tests/` — unit-тесты.

## Реализованные возможности веб-интерфейса

В веб-интерфейсе доступны все основные операции управления моделью:

1. размещение автомобиля;
2. оплата парковки;
3. добавление дополнительных услуг;
4. выдача автомобиля;
5. проверка охраны;
6. просмотр текущего состояния парковки;
7. просмотр списка машин на парковке;
8. сброс общего дохода.

## Почему выбран Flask

`Flask` выбран как простой и понятный веб-фреймворк для небольшой учебной системы:

- легко показывает архитектуру клиент-серверного приложения;
- не перегружает проект лишней инфраструктурой;
- хорошо подходит для демонстрации маршрутов, шаблонов и форм;
- позволяет быстро связать HTML-интерфейс с уже готовой бизнес-логикой.

## Как проверить работу

### CLI

```bash
python run_cli.py
```

### Web

```bash
python run_web.py
```

После запуска веб-интерфейс будет доступен по адресу:

```text
http://127.0.0.1:5000
```

## Тестирование

```bash
python -m unittest discover -s tests
```
