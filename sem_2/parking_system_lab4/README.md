# Лабораторная работа №4 — веб-интерфейс к системе управления автостоянкой

Проект представляет собой клиент-серверное веб-приложение, разработанное на базе результата лабораторной работы №1.
Предметная область — система управления автостоянкой: размещение автомобилей, оплата, дополнительные услуги,
контроль охраны и отображение состояния парковки.

## Цель работы

Изучить архитектурные приёмы построения пользовательского интерфейса и принципы создания веб-приложений.

## Выполнение требований задания

### 1. Использован веб-фреймворк
В проекте используется `Flask`.

### 2. Веб-интерфейс включает управление моделью и отображение её состояния
В веб-приложении реализованы:
- размещение автомобиля;
- оплата парковки;
- добавление услуг;
- выдача автомобиля;
- проверка информации для охраны;
- просмотр состояния парковки;
- просмотр списка автомобилей;
- сброс общего дохода.

### 3. Выделен общий код для ЛР №1 и ЛР №4
Общая логика сосредоточена в слоях:
- `app/models/`
- `app/services/`
- `app/facade.py`

Один и тот же код используется как в CLI (`run_cli.py`), так и в web (`run_web.py`).

### 4. Документирование в markdown
В проекте присутствуют:
- `README.md`
- `docs/IMPLEMENTATION.md`

### 5. Готово к размещению на GitHub
Структура проекта подготовлена для публикации в репозитории.

## Архитектура

Проект построен по принципу разделения ответственности:

- **Models** — сущности предметной области;
- **Service** — бизнес-логика управления парковкой;
- **Facade** — общий слой между интерфейсами и логикой;
- **CLI** — консольный интерфейс;
- **Web** — веб-интерфейс на Flask;
- **Templates + Static** — HTML/CSS-представление.

## Установка и запуск

```bash
pip install -r requirements.txt
```

### Запуск CLI

```bash
python run_cli.py
```

### Запуск веб-приложения

```bash
python run_web.py
```

Затем откройте в браузере:

```text
http://127.0.0.1:5000
```

## Тестирование

```bash
python -m unittest discover -s tests
```

## Что важно показать преподавателю

1. Общая бизнес-логика не зависит от интерфейса.
2. CLI и Web используют одни и те же модели и сервисы.
3. Веб-приложение работает по клиент-серверному принципу: браузер отправляет HTTP-запросы, сервер обрабатывает их и возвращает HTML-страницы.
4. Состояние модели сохраняется в JSON-файл.
5. Реализация документирована в markdown.
