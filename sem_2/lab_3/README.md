# Crimsoland

## О проекте

Данный проект представляет собой 2D top-down shooter, разработанный в Unity в рамках лабораторной работы.

Основная идея проекта — реализовать игровую систему с волнами врагов, несколькими видами оружия, разными типами противников, пользовательским интерфейсом, сохранением рекорда и базовым звуковым сопровождением.

Проект построен как набор отдельных игровых сцен и скриптовых модулей, каждый из которых отвечает за свою часть логики.

## Описание реализации

## 1. Система управления

Для проекта используется **Unity Input System**.  
Ввод реализован через отдельный asset с набором actions.

Используются следующие действия:
- движение игрока
- выстрел
- выбор оружия 1
- выбор оружия 2
- выбор оружия 3

Такой подход позволил:
- отказаться от старого ввода Unity
- избежать конфликтов между Input Manager и Input System
- централизованно управлять пользовательским вводом

---

## 2. Игрок

Игрок реализован как объект со следующими подсистемами:

### Движение
Отвечает за перемещение игрока по арене с учетом направления ввода.  
Движение ограничено пределами экрана, чтобы объект не уходил за границы игровой области.

### Поворот к курсору
Игрок всегда поворачивается в сторону курсора мыши.  
Это реализовано через перевод позиции мыши из экранных координат в мировые и расчет угла поворота.

### Точка выстрела
У игрока есть отдельный дочерний объект `FirePoint`, из которого создаются пули.  
Это позволяет:
- контролировать место появления пули
- удобно масштабировать систему оружия
- визуально выделить направление стрельбы

### Здоровье игрока
Игрок имеет:
- максимальное здоровье
- текущее здоровье
- получение урона при касании врага
- смерть при потере всех HP

При смерти:
- показывается надпись `Game Over`
- появляется кнопка возврата в меню
- игра останавливается

---

## 3. Система оружия

В игре реализовано 3 разных типа оружия.

### Glock
Одиночный выстрел.  
Используется как базовое оружие с обычной скорострельностью.

### Shotgun
Двойной выстрел под углом.  
Создает две пули, разлетающиеся в разные стороны относительно направления игрока.

### Rifle
Очередь из трех выстрелов.  
Реализована через корутину, которая создает три отдельные пули с небольшой задержкой между ними.

### Переключение оружия
Переключение оружия осуществляется через отдельные input actions.  
Текущее выбранное оружие отображается в HUD.

---

## 4. Пули

Пуля реализована как отдельный prefab.

Функции пули:
- движение в заданном направлении
- ограниченное время жизни
- нанесение урона врагу при попадании
- уничтожение после столкновения

Пуля не убивает врага напрямую, а передает урон в метод получения урона врага.  
Это позволило перейти от примитивной логики "попал = убил" к полноценной системе HP врагов.

---

## 5. Система врагов

Каждый враг реализован на базе общего скрипта и имеет набор параметров:

- скорость движения
- максимальное здоровье
- текущее здоровье
- контактный урон игроку
- задержку между ударами
- ценность по очкам

### Поведение врагов
В базовой реализации враг:
- ищет игрока
- движется в его сторону
- наносит урон при контакте
- получает урон от пуль
- умирает при потере HP
- сообщает `WaveManager`, что был уничтожен

### Типы врагов
В проекте реализовано 5 типов врагов:
- `Basic`
- `Fast`
- `Tank`
- `Heavy`
- `Swift`

Они отличаются:
- параметрами скорости
- количеством здоровья
- контактным уроном
- количеством очков за уничтожение
- внешним видом

---

## 6. Система волн

Одна из ключевых частей проекта — система волн врагов.

### Внешний конфиг
Все волны описаны во внешнем файле `waves.json`.

Для каждой волны задаются:
- номер волны
- тип врага
- количество врагов
- задержка между появлением врагов

### WaveManager
Для управления волнами реализован отдельный скрипт `WaveManager`, который:
- загружает конфиг из JSON
- считывает массив волн
- запускает первую волну
- спавнит врагов нужного типа
- отслеживает количество живых врагов
- после уничтожения всех врагов запускает следующую волну

В проекте реализовано **20 волн**, как требуется по условию лабораторной работы.

## 7. HUD

Игровой интерфейс реализован через `Canvas` и `TextMeshPro`.

В HUD отображаются:
- текущая волна
- текущее оружие
- текущее количество очков
- здоровье игрока

Для работы с интерфейсом выделен отдельный скрипт `GameHUD`, который централизованно обновляет значения на экране.

---

## 8. Система очков и рекордов

### Очки
Игрок получает очки за уничтожение врагов.  
Количество очков зависит от параметра `scoreValue` конкретного врага.

### Рекорд
Лучший результат сохраняется между запусками игры.  
Для этого используется `PlayerPrefs`.

Рекорд обновляется в момент смерти игрока:
- если текущий результат выше сохраненного
- новый рекорд записывается

На сцене `Records` реализован отдельный экран, который отображает лучший сохраненный результат.

## 10. Звуковое сопровождение

В проекте подключены:
- звук выстрела
- звук смерти врага
- звук получения урона игроком
- звук `Game Over`
- фоновая музыка на игровой сцене

Для этого используются `AudioSource` и `AudioClip`.
Музыка запускается отдельным объектом на сцене `Game`, а эффекты проигрываются из соответствующих игровых объектов.

## 11. Основные скрипты проекта

### Игрок
- `PlayerController.cs` — движение игрока
- `PlayerAim.cs` — поворот игрока к курсору
- `PlayerShooter.cs` — стрельба и переключение оружия
- `PlayerHealth.cs` — здоровье игрока и смерть

### Боевая система
- `Bullet.cs` — логика пули
- `Enemy.cs` — логика врага, HP, контактный урон, смерть

### Волны
- `WaveData.cs` — структуры данных для JSON
- `WaveManager.cs` — загрузка и запуск волн

### Интерфейс
- `GameHUD.cs` — отображение игровой информации
- `RecordsScreen.cs` — вывод лучшего результата

### Сохранение данных
- `RecordManager.cs` — сохранение и загрузка рекорда

### Сцены
- `SceneLoader.cs` — переходы между сценами
